# 文件管理模块使用说明

## 功能概述

文件管理模块是一个用于处理 Word 文档的工具，主要功能包括：

1. **上传多个 Word 文档** - 支持 .doc 和 .docx 格式
2. **合并文档内容** - 将多个文档内容整合到一个文件
3. **导出选项** - 可导出为 TXT 文本文件或 Word 文档（RTF 格式）

## 使用方法

### 1. 选择文件

有两种方式添加 Word 文档：

- **点击选择**：点击"📂 选择 Word 文档"按钮或点击上传区域
- **拖拽上传**：直接将 Word 文件拖拽到上传区域

### 2. 管理文件列表

- 已选文件会显示在列表中，包含文件名和大小信息
- 点击每个文件旁边的"移除"按钮可以删除单个文件
- 点击"清空列表"按钮可以清空所有文件

### 3. 合并选项

在合并前，可以选择以下选项：

- **合并时添加文件名标题**：在每个文档内容前添加文件名作为标题
- **文件间添加分隔符**：在不同文档内容之间添加分隔线

### 4. 导出文件

- **合并为 TXT**：将所有文档内容合并为一个纯文本文件（.txt）
- **合并为 Word**：将所有文档内容合并为一个 Word 兼容文档（.rtf 格式）

合并后的内容会在"合并预览/内容输出"区域显示，同时自动下载合并后的文件。

## 技术特性

### DOCX 文件解析

- 使用自定义的 ZIP 解析器提取 DOCX 文件内容
- 解析 Word XML 格式，保留段落结构
- 支持多种编码格式
- 包含多层降级策略，确保最大兼容性

### 文件格式支持

- **.docx** - 完全支持，可准确提取文本内容
- **.doc** - 基础支持（建议转换为 .docx 以获得更好效果）
- **输出格式**：
  - TXT - 纯文本格式
  - RTF - Rich Text Format，可被 Microsoft Word 打开

### 浏览器兼容性

- 支持现代浏览器（Chrome, Edge, Firefox, Safari）
- 使用标准 Web API，无需额外插件
- 文件处理完全在浏览器本地完成，保护隐私

## 注意事项

1. **文件大小限制**：建议单个文件不超过 10MB，以确保流畅的处理速度
2. **格式建议**：推荐使用 .docx 格式以获得最佳文本提取效果
3. **复杂格式**：表格、图片等复杂格式会被转换为纯文本
4. **隐私安全**：所有文件处理都在本地浏览器中完成，不会上传到服务器

## 常见问题

### Q: 为什么有些文档内容提取不完整？

A: 可能原因：
- 文档使用了复杂的格式（如文本框、图形）
- 文档是旧版 .doc 格式（建议转换为 .docx）
- 文档包含加密或保护

### Q: 合并后的 Word 文档格式丢失了？

A: 导出的 RTF 格式是简化的 Word 格式，只保留纯文本内容。如需保留完整格式，建议：
- 先导出为 TXT
- 在 Word 中手动打开并调整格式

### Q: 可以处理多少个文件？

A: 理论上没有数量限制，但建议：
- 一次处理不超过 20 个文件
- 总大小不超过 50MB
- 以确保浏览器性能
