# 分隔符功能修复说明

## ✅ 问题修复

已修复分隔符没有另起一行的问题。现在所有分隔符都会在新的一行开始显示。

## 📋 修复后的效果

### 1. 单行空白
**之前（错误）**：
```
文件1内容文件2内容
```

**现在（正确）**：
```
文件1内容

文件2内容
```
- 文件1内容结束后换行
- 再加一个空行
- 然后是文件2内容

---

### 2. 双行空白
**之前（错误）**：
```
文件1内容
文件2内容
```

**现在（正确）**：
```
文件1内容


文件2内容
```
- 文件1内容结束后换行
- 再加两个空行
- 然后是文件2内容

---

### 3. 分隔线
**效果**：
```
文件1内容
------------------------------------------------------------
文件2内容
```
- 文件1内容结束后换行
- 显示分隔线
- 再换行后是文件2内容

---

### 4. 星号分隔
**效果**：
```
文件1内容
************************************************************
文件2内容
```
- 文件1内容结束后换行
- 显示星号线
- 再换行后是文件2内容

---

### 5. 自定义
**示例 1 - 简单文本**：
```
输入：===
效果：
文件1内容
===
文件2内容
```

**示例 2 - 带换行的自定义**：
```
输入：\n【分隔】\n
效果：
文件1内容

【分隔】

文件2内容
```

**说明**：
- 如果自定义内容没有以 `\n` 开头，系统会自动在前面添加一个换行
- 这确保分隔符总是在新的一行开始

## 🔧 技术细节

### 修改内容

```javascript
// 修复前
case 'newline':
    return '\n';  // 只有一个换行

// 修复后
case 'newline':
    return '\n\n';  // 先换行，再加空行
```

### 完整逻辑

```javascript
function getSeparator() {
    switch (separatorType) {
        case 'newline':
            // 单行空白：先换行，再加一个空行
            return '\n\n';
            
        case 'double-newline':
            // 双行空白：先换行，再加两个空行
            return '\n\n\n';
            
        case 'line':
            // 分隔线：先换行，加分隔线，再换行
            return '\n' + '-'.repeat(60) + '\n';
            
        case 'stars':
            // 星号分隔：先换行，加星号线，再换行
            return '\n' + '*'.repeat(60) + '\n';
            
        case 'custom':
            let customValue = /* 处理转义字符 */;
            
            // 确保自定义分隔符前面有换行
            if (!customValue.startsWith('\n')) {
                customValue = '\n' + customValue;
            }
            
            return customValue;
    }
}
```

## ✨ 改进说明

### 1. 自动换行保护
- 所有分隔符都确保在新行开始
- 自定义分隔符会自动添加前导换行（如果没有的话）

### 2. 更符合直觉
- "单行空白"真正产生一个空行
- "双行空白"真正产生两个空行
- 分隔线和星号线都在独立的行上

### 3. 自定义更灵活
- 用户可以完全控制分隔内容
- 系统会智能添加必要的换行
- 支持复杂的多行分隔

## ⚠️ 注意事项

1. **换行符说明**
   - `\n` 在输入框中需要输入反斜杠 + n
   - 系统会自动转换为真正的换行符

2. **自动换行**
   - 如果自定义内容不以 `\n` 开头
   - 系统会自动在前面加一个换行
   - 确保分隔符在新行开始

3. **预览确认**
   - 合并后先查看预览区域
   - 确认分隔效果符合预期
   - 再下载文件
