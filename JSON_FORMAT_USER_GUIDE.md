# JSON 格式化功能使用指南

## 新功能概述

JSON 格式化模块现在支持：
1. ✅ **智能错误提示** - 显示详细的 JSON 格式错误信息
2. ✅ **自动去除转义符** - 自动处理带转义符的 JSON 字符串

## 使用场景

### 场景 1：格式化普通 JSON

**输入示例：**
```json
{"name":"John","age":30,"city":"Beijing"}
```

**操作：** 直接粘贴到输入框

**结果：** 自动格式化为易读的格式
```json
{
  "name": "John",
  "age": 30,
  "city": "Beijing"
}
```

---

### 场景 2：处理带转义符的 JSON（包括嵌套转义）

**常见来源：**
- API 响应中的 JSON 字符串
- 日志文件中的 JSON 数据
- 数据库中存储的 JSON 字符串
- **嵌套场景：** 数组或对象中某些字段值是转义的 JSON 字符串

**输入示例 1（简单转义）：**
```
"{\"user\":{\"name\":\"Alice\",\"email\":\"alice@example.com\"},\"status\":\"active\"}"
```

**操作：** 直接粘贴到输入框

**结果：** 
1. 自动移除转义符
2. 输入框更新为：`{"user":{"name":"Alice","email":"alice@example.com"},"status":"active"}`
3. 输出框显示格式化后的 JSON
4. 提示消息：`JSON 已格式化（已自动移除转义符）`

---

**输入示例 2（嵌套转义 - 实际场景）：**
```json
[{
  "tag": "137",
  "msg_id": "88550786847158120670",
  "data": "{\"audit_status\":3,\"name\":\"test\",\"video_info\":{\"duration\":10,\"format\":\"mp4\"}}"
}]
```

**操作：** 直接粘贴到输入框

**结果：**
1. 自动识别 `data` 字段是转义的 JSON 字符串
2. 递归解析所有嵌套的转义 JSON
3. 输入框更新为完全解析的 JSON
4. 输出框显示：
```json
[
  {
    "tag": "137",
    "msg_id": "88550786847158120670",
    "data": {
      "audit_status": 3,
      "name": "test",
      "video_info": {
        "duration": 10,
        "format": "mp4"
      }
    }
  }
]
```
5. 提示消息：`JSON 已格式化（已自动移除转义符）`

**💡 智能特性：**
- ✅ 自动检测任意深度的嵌套转义
- ✅ 支持数组中的转义 JSON
- ✅ 支持对象属性中的转义 JSON
- ✅ 递归处理多层嵌套

---

### 场景 3：发现 JSON 格式错误

**输入示例（缺少引号）：**
```
{name: "test", value: 123}
```

**结果：**
- ❌ 输出框清空
- 🔴 显示错误：`JSON 格式错误: Unexpected token n in JSON at position 1`
- 💡 根据错误信息定位问题

**常见错误类型：**

| 错误类型 | 示例 | 错误提示 |
|---------|------|---------|
| 缺少引号 | `{name: "test"}` | `Unexpected token n` |
| 多余逗号 | `{"name": "test",}` | `Unexpected token }` |
| 缺少逗号 | `{"a": 1 "b": 2}` | `Unexpected string` |
| 括号不匹配 | `{"arr": [1, 2}` | `Unexpected token }` |

## 功能按钮说明

### 格式化按钮
- **功能：** 手动触发格式化（通常会自动格式化）
- **快捷键：** 无
- **提示：** 输入后会自动格式化，无需手动点击

### 压缩按钮
- **功能：** 将格式化的 JSON 压缩为单行
- **用途：** 适合复制到代码或配置文件中
- **示例：** `{"name":"test","value":123}`

### 复制输出按钮
- **功能：** 一键复制输出框中的内容
- **提示：** 格式化后的 JSON 会保留缩进

### 清空按钮
- **功能：** 清空输入框和输出框
- **快捷方式：** 直接清空输入框也会自动清空输出

---

## 高级功能

### 排序键（Sort Keys）
- **位置：** 输入框下方的复选框
- **功能：** 按字母顺序排序 JSON 对象的键
- **用途：** 便于比较两个 JSON 对象

**示例：**
```json
// 原始
{"z": 1, "a": 2, "m": 3}

// 排序后
{"a": 2, "m": 3, "z": 1}
```

### 缩进设置
- **位置：** 输入框下方的数字输入框
- **默认值：** 2 个空格
- **可选值：** 2, 4, 8 等
- **用途：** 自定义格式化后的缩进大小

## 实用技巧

### 技巧 1：快速验证 JSON
复制任何 JSON 数据，粘贴到输入框：
- ✅ 如果格式正确，会自动格式化
- ❌ 如果格式错误，会显示详细错误

### 技巧 2：处理 API 响应
从浏览器开发者工具或 Postman 复制的 JSON 响应，可能包含转义符：
```
"{\"code\":200,\"data\":{\"id\":1}}"
```
直接粘贴即可自动处理！

### 技巧 3：格式化日志中的 JSON
日志文件中的 JSON 通常是转义的：
```
[2024-01-01 10:00:00] Response: "{\"status\":\"success\",\"message\":\"OK\"}"
```
只需复制 JSON 部分（包括外层引号）即可自动处理。

### 技巧 4：点击复制 JSON 值
在格式化后的输出中：
- 点击任何键名可以复制对应的值
- 支持复制整个对象或数组
- 支持下载和删除操作

## 常见问题

### Q1: 为什么我的 JSON 无法格式化？
**A:** 检查错误提示信息，常见问题：
- 缺少引号（键名必须用双引号）
- 多余或缺少逗号
- 括号不匹配
- 使用了单引号（JSON 只支持双引号）

### Q2: 转义符没有自动移除？
**A:** 确保：
- JSON 字符串以 `"` 开始和结束
- 内部是有效的 JSON 格式
- 如果还是不行，尝试手动移除外层引号

### Q3: 输出框显示的是旧内容？
**A:** 这个问题已修复：
- 错误时会自动清空输出框
- 如果仍有问题，点击"清空"按钮

### Q4: 如何复制压缩后的 JSON？
**A:** 
1. 点击"压缩"按钮
2. 点击"复制输出"按钮
3. 或直接从输出框中选择复制
