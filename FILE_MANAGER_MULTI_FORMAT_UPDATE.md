# 文件管理模块更新说明

## 更新日期
2026-02-17

## 更新内容

### 功能扩展：支持多种文件类型

文件管理模块已从仅支持 Word 文档扩展为支持多种常见文本文件格式。

### 支持的文件类型

现在支持以下文件格式：

1. **Word 文档**
   - `.doc` - Word 97-2003 文档
   - `.docx` - Word 2007+ 文档

2. **纯文本文件**
   - `.txt` - 纯文本文件

3. **Markdown 文件**
   - `.md` - Markdown 文档
   - `.markdown` - Markdown 文档

4. **HTML 文件**
   - `.html` - HTML 网页文件
   - `.htm` - HTML 网页文件

### 文件图标

每种文件类型都有对应的图标显示：
- 📘 `.docx` - Word 2007+ 文档
- 📗 `.doc` - Word 97-2003 文档
- 📝 `.txt` - 纯文本文件
- 📋 `.md` / `.markdown` - Markdown 文档
- 🌐 `.html` / `.htm` - HTML 文件
- 📄 其他文件类型

### 技术实现

#### 1. HTML 更新
- 更新文件选择器的 `accept` 属性，支持多种文件扩展名
- 修改界面提示文本，说明支持的文件类型
- 更新按钮和提示文字，从"Word 文档"改为通用的"文件"

#### 2. JavaScript 更新

**新增函数：**
- `getFileIcon(fileName)` - 根据文件扩展名返回对应的图标
- `readTextFile(file)` - 读取纯文本文件（txt、md、html）
- `readFileContent(file)` - 统一的文件读取接口，根据文件类型调用相应的读取函数

**修改函数：**
- `handleFiles(files)` - 更新文件验证逻辑，支持多种文件扩展名
- `updateFileList()` - 使用 `getFileIcon()` 函数显示不同的文件图标
- `mergeToTxt()` - 使用 `readFileContent()` 替代 `readWordDocument()`
- `mergeToWord()` - 使用 `readFileContent()` 替代 `readWordDocument()`

### 使用方法

1. **选择文件**
   - 点击"📂 选择文件"按钮
   - 或直接拖拽文件到上传区域

2. **支持的操作**
   - 合并为 TXT：将所有文件内容合并为一个 .txt 文件
   - 合并为 Word：将所有文件内容合并为一个 .docx 文件
   - 清空列表：移除所有已选文件

3. **分隔符设置**
   - 单行空白（默认）
   - 双行空白
   - 分隔线
   - 星号分隔
   - 自定义分隔符

### 文件读取说明

- **纯文本文件** (txt, md, html)：使用 UTF-8 编码直接读取
- **Word 文档** (docx)：使用 XML 解析提取文本内容
- **Word 文档** (doc)：由于格式复杂，建议转换为 .docx 格式

### 注意事项

1. 所有文件都会被读取为纯文本内容
2. Word 文档会尝试提取文本，但可能丢失格式信息
3. HTML 文件会保留原始 HTML 标签
4. 建议使用 UTF-8 编码的文本文件以确保正确显示中文

### 兼容性

- 完全向后兼容，原有的 Word 文档功能保持不变
- 新增的文件类型无需额外配置即可使用
- 所有现代浏览器均支持
