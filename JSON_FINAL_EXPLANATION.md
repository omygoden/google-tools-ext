# JSON 格式化模块 - 最终说明

## ✅ 实现的功能

### 1. 详细错误信息提示
- 显示具体的 JSON 格式错误信息
- 显示错误位置的上下文
- 自动清理 BOM 和不可见字符

### 2. 自动移除最外层转义符

**重要说明：** 只处理最外层的转义，**不会**递归处理内部字符串值。

---

## 📋 工作原理

### 场景 A：整个输入是转义的 JSON 字符串

**输入：**
```
"{\"name\":\"John\",\"age\":30}"
```

**处理：**
1. 检测到以 `"` 开始和结束
2. 移除外层引号和转义符
3. 得到：`{"name":"John","age":30}`

**输出：**
```json
{
  "name": "John",
  "age": 30
}
```

---

### 场景 B：JSON 内部包含转义的字符串值（保持原样）

**输入：**
```json
[{
  "tag": "137",
  "data": "{\"status\":3,\"name\":\"test\"}"
}]
```

**处理：**
1. 直接解析成功（因为这本身就是有效的 JSON）
2. `data` 字段的值是一个**字符串**
3. **不会**自动将这个字符串解析为 JSON 对象

**输出：**
```json
[
  {
    "tag": "137",
    "data": "{\"status\":3,\"name\":\"test\"}"
  }
]
```

**注意：** `data` 的值仍然是字符串，不是对象！

---

## 🎯 使用场景

### ✅ 适用场景

1. **从日志复制的转义 JSON**
   ```
   输入："{\"error\":\"not found\"}"
   输出：{"error":"not found"}
   ```

2. **API 返回的转义 JSON 字符串**
   ```
   输入："{\"code\":200,\"data\":{}}"
   输出：{"code":200,"data":{}}
   ```

3. **包含转义符的完整 JSON 字符串**
   ```
   输入："{\"users\":[{\"id\":1}]}"
   输出：{"users":[{"id":1}]}
   ```

### ❌ 不适用场景

如果您希望将 JSON 内部的字符串值也解析为 JSON 对象，这个工具**不会**自动处理。

**示例：**
```json
// 输入
{"data": "{\"nested\":true}"}

// 输出（data 仍然是字符串）
{
  "data": "{\"nested\":true}"
}

// 如果您希望得到这样的结果，需要手动处理：
{
  "data": {
    "nested": true
  }
}
```

## 🔧 手动处理嵌套转义

如果您需要将内部的转义字符串也解析为 JSON，可以：

### 方法 1：两次格式化

1. 第一次格式化得到：
   ```json
   {
     "data": "{\"nested\":true}"
   }
   ```

2. 复制 `data` 的值：`"{\"nested\":true}"`

3. 再次粘贴到输入框，自动去除转义：
   ```json
   {
     "nested": true
   }
   ```

### 方法 2：使用浏览器控制台

```javascript
const input = `{"data": "{\"nested\":true}"}`;
const parsed = JSON.parse(input);
parsed.data = JSON.parse(parsed.data);
console.log(JSON.stringify(parsed, null, 2));
```

## 💡 总结

**核心原则：**
- ✅ 只处理最外层的转义
- ✅ 保持 JSON 内部字符串值的原样
- ✅ 提供清晰的错误信息

**设计理念：**
这样设计是为了保持数据的原始结构。如果自动递归解析所有转义字符串，可能会改变数据的原始含义（因为某些字段的值就是应该是字符串，而不是对象）。

现在工具的行为更加可预测和安全！✨
