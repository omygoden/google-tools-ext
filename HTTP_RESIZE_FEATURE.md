# HTTP 请求模块 - 放大/缩小功能说明

## 功能概述

HTTP 请求模块现在支持动态调整请求配置区域和响应结果区域的大小，让用户可以根据需要灵活调整界面布局。

## 使用方法

### 请求配置区域（Request Config）

在请求配置区域的标题栏右侧，有两个控制按钮：

- **▲ 按钮**：放大请求配置区域
  - 点击后，请求配置区域会变大，响应区域会相应缩小
  - 适合需要查看或编辑大量请求参数、Headers 或 Body 时使用

- **▼ 按钮**：缩小请求配置区域
  - 点击后，请求配置区域会变小，响应区域会相应放大
  - 适合请求配置简单，需要更多空间查看响应结果时使用

### 响应结果区域（Response Section）

在响应结果区域的标题栏右侧（状态码旁边），同样有两个控制按钮：

- **▲ 按钮**：放大响应结果区域
  - 点击后，响应区域会变大，请求配置区域会相应缩小
  - 适合需要查看大量响应数据时使用

- **▼ 按钮**：缩小响应结果区域
  - 点击后，响应区域会变小，请求配置区域会相应放大
  - 适合响应数据较少，需要更多空间配置请求时使用

## 技术特性

### 平滑过渡动画
- 所有大小调整都带有 0.3 秒的平滑过渡动画
- 提供流畅的用户体验

### 智能平衡
- 放大一个区域时，另一个区域会自动缩小以保持平衡
- 缩小一个区域时，另一个区域会自动放大
- 确保两个区域始终可见且可用

### 尺寸限制
- **最小 flex 值**：0.5（防止区域完全消失）
- **最大 flex 值**：6（防止区域过大）
- **调整步长**：0.5（每次点击的调整幅度）

### 视觉反馈
- 每次调整后，底部会显示提示消息
- 例如："请求配置区域已放大"、"响应结果区域已缩小"

## 使用场景示例

### 场景 1：配置复杂的 POST 请求
1. 需要添加多个 Headers
2. Body 中有大量 JSON 数据
3. **操作**：点击请求区域的 ▲ 按钮 2-3 次，获得更大的编辑空间

### 场景 2：查看大量响应数据
1. 请求配置简单（只有 URL）
2. 响应返回大量 JSON 数据
3. **操作**：点击响应区域的 ▲ 按钮 2-3 次，获得更大的查看空间

### 场景 3：快速测试 API
1. 简单的 GET 请求
2. 需要快速查看响应
3. **操作**：点击请求区域的 ▼ 按钮，缩小请求区域，专注于响应

## 键盘快捷键（未来可能添加）

目前通过鼠标点击按钮操作，未来可以考虑添加：
- `Ctrl/Cmd + ↑`：放大当前焦点区域
- `Ctrl/Cmd + ↓`：缩小当前焦点区域
- `Ctrl/Cmd + 0`：重置为默认大小

## 注意事项

1. **最小尺寸限制**：区域不会完全消失，始终保持最小可见尺寸
2. **最大尺寸限制**：区域不会无限放大，有合理的上限
3. **状态不保存**：刷新页面后会恢复默认大小（未来可以添加保存功能）
