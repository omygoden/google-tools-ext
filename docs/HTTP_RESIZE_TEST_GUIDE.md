# HTTP 请求模块放大/缩小功能 - 快速测试指南

## 测试步骤

### 1. 重新加载扩展
1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions`
3. 找到 "SQL/JSON → Go Struct (Local)" 扩展
4. 点击刷新按钮 🔄

### 2. 打开扩展
1. 点击扩展图标
2. 切换到 "HTTP 请求" 标签（🚀 HTTP 请求）

### 3. 测试请求区域放大/缩小

#### 测试放大功能
1. 在请求配置区域的标题栏右侧，找到 ▲ 和 ▼ 按钮
2. 点击 **▲ 按钮** 3 次
3. **预期结果**：
   - 请求配置区域（Params/Headers/Body）逐渐变大
   - 响应结果区域逐渐变小
   - 底部显示提示："请求配置区域已放大"
   - 过渡动画平滑（0.3秒）

#### 测试缩小功能
1. 点击 **▼ 按钮** 3 次
2. **预期结果**：
   - 请求配置区域逐渐变小
   - 响应结果区域逐渐变大
   - 底部显示提示："请求配置区域已缩小"

### 4. 测试响应区域放大/缩小

#### 测试放大功能
1. 在响应结果区域的标题栏右侧（"响应结果"文字旁边），找到 ▲ 和 ▼ 按钮
2. 点击 **▲ 按钮** 3 次
3. **预期结果**：
   - 响应结果区域逐渐变大
   - 请求配置区域逐渐变小
   - 底部显示提示："响应结果区域已放大"

#### 测试缩小功能
1. 点击 **▼ 按钮** 3 次
2. **预期结果**：
   - 响应结果区域逐渐变小
   - 请求配置区域逐渐变大
   - 底部显示提示："响应结果区域已缩小"

### 5. 测试边界情况

#### 测试最大尺寸限制
1. 连续点击请求区域的 ▲ 按钮 15 次
2. **预期结果**：
   - 请求区域不会无限放大
   - 达到最大值后继续点击不再变化
   - 响应区域保持最小可见尺寸

#### 测试最小尺寸限制
1. 连续点击请求区域的 ▼ 按钮 15 次
2. **预期结果**：
   - 请求区域不会完全消失
   - 达到最小值后继续点击不再变化
   - 响应区域保持最大限制尺寸

### 6. 实际使用场景测试

#### 场景 1：配置复杂请求
1. 切换到 "Body" 标签
2. 选择 "json" 类型
3. 点击请求区域 ▲ 按钮 2-3 次
4. 在 JSON 输入框中粘贴大量数据
5. **验证**：有足够的空间编辑和查看 JSON

#### 场景 2：查看响应数据
1. 在 URL 输入框输入：`https://httpbin.org/get`
2. 点击 "发送" 按钮
3. 等待响应返回
4. 点击响应区域 ▲ 按钮 2-3 次
5. **验证**：响应数据显示清晰，有足够的查看空间

## 常见问题排查

### 问题 1：按钮点击无反应
**可能原因**：
- JavaScript 加载失败
- 事件监听器未绑定

**解决方案**：
1. 按 F12 打开开发者工具
2. 查看 Console 是否有错误
3. 重新加载扩展

### 问题 2：区域大小不变化
**可能原因**：
- CSS transition 未生效
- flex 值未正确应用

**解决方案**：
1. 检查元素的 style 属性
2. 查看 computed styles
3. 确认 flex 值是否更新

### 问题 3：过渡动画不流畅
**可能原因**：
- 浏览器性能问题
- CSS transition 设置不当

**解决方案**：
1. 检查 transition 属性是否正确
2. 尝试调整 transition 时长
